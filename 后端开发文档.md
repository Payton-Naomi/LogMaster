# LogMaster 后端开发文档

## 1. 后端职责

当前后端由 Go 标准库 `net/http` 实现，主要提供三类能力：

- 嵌入并托管 `static/index.html` 页面。
- 提供飞书 OAuth 登录、回调、退出和当前用户信息接口。
- 提供 `/api/stream` 服务端事件流，实时推送多个 goroutine 的运行输出。

## 2. 目录结构

```text
logmaster/
├── main.go                 # 后端入口，注册静态页面、认证接口和流式接口
├── static/
│   └── index.html          # 嵌入式页面
└── internal/
    ├── auth/               # 飞书登录、会话管理、用户信息
    ├── config/             # 环境变量配置
    ├── response/           # JSON 响应封装
    └── stream/             # 实时线程输出 SSE 接口
```

## 3. 本地运行

进入项目根目录：

```powershell
cd C:\Users\DELL\Desktop\codex\logmaster
go run .
```

启动成功后访问：

```text
http://localhost:8080
```

## 4. 环境变量

| 变量名 | 是否必填 | 说明 |
| --- | --- | --- |
| `FEISHU_APP_ID` | 否 | 飞书应用 ID，未设置时使用代码中的默认测试值 |
| `FEISHU_APP_SECRET` | 是 | 飞书应用密钥，不应写入代码或提交到仓库 |
| `FEISHU_REDIRECT_URI` | 否 | 飞书回调地址，默认 `http://localhost:8080/api/auth/callback` |

PowerShell 示例：

```powershell
$env:FEISHU_APP_ID="你的飞书 App ID"
$env:FEISHU_APP_SECRET="你的飞书 App Secret"
$env:FEISHU_REDIRECT_URI="http://localhost:8080/api/auth/callback"
go run .
```

## 5. 接口说明

### 获取飞书登录地址

```text
GET /api/auth/feishu-url
```

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "https://accounts.feishu.cn/open-apis/authen/v1/authorize?..."
  }
}
```

### 飞书 OAuth 回调

```text
GET /api/auth/callback?code=xxx&state=xxx
```

后端会校验 `state`，用 `code` 换取飞书用户访问令牌，再获取用户信息并写入 `session_token` Cookie。

### 当前用户信息

```text
GET /api/user/info
```

未登录时返回 `401`，已登录时返回当前用户信息。

### 退出登录

```text
POST /api/auth/logout
```

后端会删除内存会话并清理 `session_token` Cookie。

### 实时线程输出

```text
GET /api/stream
```

该接口要求用户已登录，响应类型为 `text/event-stream`。每条消息格式如下：

```json
{
  "box": 1,
  "text": "Version: 1\nName: Alice\nStep: 1/8\nStatus: goroutine 1 is working...\n\n",
  "done": false
}
```

## 6. 开发注意事项

- 当前会话存储在进程内存中，服务重启后登录态会失效。
- `FEISHU_APP_SECRET` 必须通过环境变量配置，不要硬编码到仓库。
- `/api/stream` 使用 SSE，反向代理部署时需要关闭代理缓冲。
- 静态页面通过 Go `embed` 打包，修改 `static/index.html` 后重新 `go run .` 或重新构建即可生效。
