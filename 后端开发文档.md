# LogMaster 后端开发文档

## 1. 后端职责

当前后端由 Go 标准库 `net/http` 实现，主要提供以下能力：

- 提供飞书 OAuth 登录、回调、退出和当前用户信息接口。
- 提供日志上传、压缩包解压、异步解析和结果查询接口。
- 提供可选的 Agent 日志诊断扩展接口。
- 提供 `/api/health` 服务健康检查。

## 2. 目录结构

```text
logmaster/
├── main.go                 # 后端 API 入口
├── frontend/               # Vue 3 + Vite 前端
└── internal/
    ├── auth/               # 飞书登录、会话管理、用户信息
    ├── config/             # 环境变量配置
    ├── database/           # PostgreSQL 连接与自动迁移
    ├── logs/               # 日志上传、解压、解析和 Agent 扩展
    ├── response/           # JSON 响应封装
```

## 3. 本地运行

进入项目根目录：

```powershell
cd C:\Users\DELL\Desktop\codex\logmaster
go run .
```

构建前端并启动融合服务：

```powershell
npm.cmd --prefix frontend run build
go run .
```

启动成功后访问：

```text
http://localhost:8080
```

日常前端开发也可以在 `frontend` 目录执行 `npm.cmd run dev`，通过 `http://localhost:3000` 使用热更新。

## 4. 环境变量

| 变量名 | 是否必填 | 说明 |
| --- | --- | --- |
| `FEISHU_APP_ID` | 否 | 飞书应用 ID，未设置时使用代码中的默认测试值 |
| `FEISHU_APP_SECRET` | 是 | 飞书应用密钥，不应写入代码或提交到仓库 |
| `FEISHU_REDIRECT_URI` | 否 | 飞书回调地址，默认 `http://localhost:8080/api/auth/callback` |
| `DATABASE_URL` | 是 | PostgreSQL 连接地址 |

PowerShell 示例：

```powershell
$env:FEISHU_APP_ID="你的飞书 App ID"
$env:FEISHU_APP_SECRET="你的飞书 App Secret"
$env:FEISHU_REDIRECT_URI="http://localhost:8080/api/auth/callback"
$env:DATABASE_URL="postgres://logmaster:logmaster@127.0.0.1:5432/logmaster?sslmode=disable"
go run .
```

## 5. 接口说明

### 获取飞书登录地址

```text
GET /api/auth/feishu-url
```

成功响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "url": "https://accounts.feishu.cn/open-apis/authen/v1/authorize?..."
  }
}
```

### 飞书 OAuth 回调

```text
GET /api/auth/callback?code=xxx&state=xxx
```

后端会校验 `state`，用 `code` 换取飞书用户访问令牌，再获取用户信息并写入 `session_token` Cookie。

### 当前用户信息

```text
GET /api/user/info
```

未登录时返回 `401`，已登录时返回当前用户信息。

### 退出登录

```text
POST /api/auth/logout
```

后端会删除内存会话并清理 `session_token` Cookie。

### 服务健康检查

```text
GET /api/health
```

```json
{
  "code": 0,
  "message": "success",
  "data": { "status": "ok" }
}
```

## 6. 开发注意事项

- 当前会话存储在进程内存中，服务重启后登录态会失效。
- `FEISHU_APP_SECRET` 必须通过环境变量配置，不要硬编码到仓库。
- 单服务模式下，8080 同时提供 `frontend/dist` 页面和 `/api` 接口；修改前端源码后需要重新执行前端构建。
