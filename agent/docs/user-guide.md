# LogCollector 用户手册

## 启动前

1. 确认 Windows 已安装 Microsoft Edge WebView2 Evergreen Runtime。
2. 将 `config_template.yaml` 复制为现场配置，填写后端地址、项目名、版本、设备编号和串口参数。
3. 前四路设备用于 V1 验收；第 5 至 8 路保持禁用，直至完成扩展验证。
4. 确认每个设备使用唯一的 `device_sn`、COM 端口或稳定的 VID/PID、USB 序列号、物理位置组合。

## 日常操作

启动 `LogCollector.exe` 后，先扫描端口，核对设备名称与物理串口，再连接单路或执行批量连接。开始任务后，设备状态应依次进入“已连接”和“采集中”。实时日志可暂停显示、搜索和按级别过滤；暂停只影响界面，不停止采集、落盘或上传。

停止任务前确认设备状态和待上传数量。正常退出会封口当前临时日志并保留 SQLite 队列。不要直接删除数据目录或未上传文件。

## 状态含义

| 状态 | 含义 | 操作 |
| --- | --- | --- |
| `pending` | 等待上传或可安全重试 | 等待自动上传，必要时检查网络 |
| `uploading` | 请求正在发送 | 不要移动对应文件 |
| `uploaded` | 后端已明确确认 | 可按保留策略清理 |
| `uncertain` | 请求可能已送达但响应未知 | 先在平台核对，再人工重试或确认 |
| `dead` | 确定失败，自动重试停止 | 修复配置或数据问题后人工处理 |

## 关闭与升级

先停止任务并关闭客户端。配置和数据目录必须独立于 EXE；升级时只替换 `LogCollector.exe`，不得覆盖 SQLite、spool 或日志目录。升级后先做单路短测，再恢复四路采集。

## 获取支持材料

问题发生后保留发生时间、设备编号、串口、客户端日志、队列状态、相关 `.log` 文件 SHA-256 和界面截图。按 `troubleshooting.md` 收集信息，不要在工单中提交 Token 或 API Key。
